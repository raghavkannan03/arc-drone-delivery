# DD-Nav2 Plan

There are two parts: 

1. Creating a program to convert a gps point inputted by the website(from where the user needs the drone to fly to either pick up the food or to deliver the picked-up food to them) 
2. Creating a program to start the navigation stack(to activate the behavior tree which controls all of the operations of what the drone does/tells it what it needs to do at all times)

For 1:

- For creating our own program to convert a GPS waypoint into a point in the global costmap, we need to determine the position of the goal relative to the origin.
    - The most straight forward manner in which to find the goal position is to save the GPS location of the origin of the costmap, then create a program that uses that finds the distance in meters between the two GPS points. The only caveat is researching to determine how the Nav2 algorithms work if the drone does not start at the origin(as the origin in this case would be fixed). We just need to confirm no problems would arise
    - For this method we assume we know the GPS location of the origin of the global costmap, which is a valid assumption to make as we should know it as the costmap will be mapped and created once(dynamic obstacles will be added on top of the base costmap as the drone flys around).
    
    Here is a sample program that ChatGPT generated that should generally work:
    
    ```jsx
    import rospy
    from geopy.distance import geodesic
    from geometry_msgs.msg import PoseStamped
    
    # Replace these with the GPS coordinates of the map's origin
    MAP_ORIGIN_LAT = 37.7749  # Latitude of the map's (0,0) point
    MAP_ORIGIN_LON = -122.4194  # Longitude of the map's (0,0) point
    
    def gps_to_map(lat, lon):
        """Convert a GPS coordinate (lat, lon) to map frame (x, y) in meters."""
        # Step 1: Compute the distance in meters from the map origin
        origin = (MAP_ORIGIN_LAT, MAP_ORIGIN_LON)
        target = (lat, lon)
    
        # Use geopy to get the distance in meters (for both x and y)
        distance_x = geodesic(origin, (lat, MAP_ORIGIN_LON)).meters
        distance_y = geodesic(origin, (MAP_ORIGIN_LAT, lon)).meters
    
        # Adjust signs based on direction (latitude/longitude comparison)
        if lat < MAP_ORIGIN_LAT:
            distance_y = -distance_y  # Target is south of origin
        if lon < MAP_ORIGIN_LON:
            distance_x = -distance_x  # Target is west of origin
    
        return distance_x, distance_y
    
    if __name__ == "__main__":
        rospy.init_node("gps_to_map_converter")
    
        # Example GPS coordinates (latitude, longitude) of the goal position
        goal_latitude = 37.7750
        goal_longitude = -122.4180
    
        # Convert GPS to map coordinates
        x, y = gps_to_map(goal_latitude, goal_longitude)
        rospy.loginfo(f"Map coordinates: x={x}, y={y}")
    ```

**Steps to Finish Nav2**

1. Creating ROS2 Subscriber to take in GPS waypoint of type NavSatFix: [https://docs.ros2.org/latest/api/sensor_msgs/msg/NavSatFix.html](https://docs.ros2.org/latest/api/sensor_msgs/msg/NavSatFix.html)
2. Keep GPS -> ECEF code
3. Subtract ECEF point from origin of costmap(in ECEF)
4. Make subtracted ECEF point into a PoseStamped Message: [https://docs.ros2.org/foxy/api/geometry_msgs/msg/PoseStamped.html](https://docs.ros2.org/foxy/api/geometry_msgs/msg/PoseStamped.html)
5. Create an action client of type NavigateToPose to start navigation server: [https://docs.ros.org/en/foxy/Tutorials/Intermediate/Writing-an-Action-Server-Client/Cpp.html](https://docs.ros.org/en/foxy/Tutorials/Intermediate/Writing-an-Action-Server-Client/Cpp.html) & [https://github.com/ros-navigation/navigation2/blob/main/nav2_msgs/action/NavigateToPose.action](https://github.com/ros-navigation/navigation2/blob/main/nav2_msgs/action/NavigateToPose.action)
