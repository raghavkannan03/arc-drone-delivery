- To visualize the lidar point clouds(both the simple and accumulated 3d map point clouds) in Rviz, you must utilize the transform from the Lidar coordinate frame to the map coordinate frame(the frame of the global world) created by the FAST-LIO algorithm(as that transforms tells where the lidar is in terms of the global map); however, the current transformation chain ends with the odom coordinate frame, which is the local map frame(currently we have a chain from lidar, to base link, to base footprint, to odom, which can be seen through a visualization in the DD_Nav_WS page in Notion). Thus, to finish the transformation chain and unify the two location measurements, we need to create a transform from the odom to map coordinate frames. To create this transformation we need to do the following calculation: (Tmap→lidar)*(Todom→lidar)^-1 where each T is the matrix that transforms a point from one coordinate frame to another(basis of linear algebra for those unfamiliar). The reason this works as taking the inverse is basically like flipping the transform around so it ends up as Tmap→lidar * Tlidar→odom, and in matrix multiplication the inner terms(the lidar) cancel out, so the product is Tmap→odom.

A code that ChatGPT generated to do this is below(it seems generally right and is a good jumping off point):

```python
import tf2_ros
import rospy
import numpy as np
from tf.transformations import concatenate_matrices, inverse_matrix, translation_matrix, quaternion_matrix

def lookup_transform(buffer, from_frame, to_frame):
    """Get the transform between two frames."""
    try:
        trans = buffer.lookup_transform(to_frame, from_frame, rospy.Time(0), rospy.Duration(1.0))
        translation = [trans.transform.translation.x, 
                       trans.transform.translation.y, 
                       trans.transform.translation.z]
        rotation = [trans.transform.rotation.x, 
                    trans.transform.rotation.y, 
                    trans.transform.rotation.z, 
                    trans.transform.rotation.w]
        return translation, rotation
    except Exception as e:
        rospy.logerr(f"Could not find transform: {e}")
        return None, None

rospy.init_node('transform_computation')

# Create a TF2 buffer and listener
tf_buffer = tf2_ros.Buffer()
tf_listener = tf2_ros.TransformListener(tf_buffer)

# Get transforms
map_to_lidar_trans, map_to_lidar_rot = lookup_transform(tf_buffer, 'lidar_frame', 'map')
odom_to_lidar_trans, odom_to_lidar_rot = lookup_transform(tf_buffer, 'lidar_frame', 'odom')

# Convert to transformation matrices
T_map_lidar = concatenate_matrices(translation_matrix(map_to_lidar_trans), quaternion_matrix(map_to_lidar_rot))
T_odom_lidar = concatenate_matrices(translation_matrix(odom_to_lidar_trans), quaternion_matrix(odom_to_lidar_rot))

# Compute inverse of T_odom_lidar
T_lidar_odom = inverse_matrix(T_odom_lidar)

# Compute T_map_odom
T_map_odom = np.dot(T_map_lidar, T_lidar_odom)

print("Transform from map to odom:")
print(T_map_odom)

```

Then, the last step is to create a static map of the world using the accumulated 3d point cloud(the accurate 3d map of the world) and publish it to the map topic so that it can be used by the ROS Nav2 library’s Costmap_2d to create a static map(which can then be the basis of the costmap/its static layer, with dynamic obstacles added on topic during the flight of the drone).

A visualization of what we want is below:

![static_map.jpeg](DD-LIO%20Plan/static_map.jpeg)

To do this the steps are:

1. **Subscribe to the `/lio/map`topic or whatever the FAST-LIO map topic name is** (of type `sensor_msgs/PointCloud2`).
2. **Project the 3D point cloud into a 2D grid** (e.g., XY plane or ground level)(determine between what the height limit of the drone is and ignore every point above that z level).
3. **Generate an `OccupancyGrid` message** from the 2D projection([https://docs.ros2.org/foxy/api/nav_msgs/msg/OccupancyGrid.html](https://docs.ros2.org/foxy/api/nav_msgs/msg/OccupancyGrid.html)).
4. **Publish the grid to a topic** that Nav2 uses (like `/map` or `/costmap`).

This can be created in python or C++, as it will be performed only periodically after the environment is completely mapped by the drone(but not while it is doing its deliver missions), and so time is not a concern.

### Step 3:

```python
from nav_msgs.msg import OccupancyGrid, MapMetaData

def generate_occupancy_grid(points, resolution=0.05, width=100, height=100):
    """
    Generate an OccupancyGrid from a set of 2D points.
    :param points: 2D numpy array of shape (N, 2) with X, Y coordinates.
    :param resolution: Grid resolution in meters per cell.
    :param width, height: Dimensions of the grid in cells.
    """
    # Create an empty grid initialized to -1 (unknown)
    grid_data = np.full((height, width), -1, dtype=np.int8)

    # Transform points to grid coordinates
    for (x, y) in points:
        gx = int((x / resolution) + width // 2)  # Offset to center of the grid
        gy = int((y / resolution) + height // 2)
        if 0 <= gx < width and 0 <= gy < height:
            grid_data[gy, gx] = 100  # Mark as occupied

    # Flatten the 2D grid into a 1D list
    flattened_data = grid_data.flatten().tolist()

    # Create the OccupancyGrid message
    grid = OccupancyGrid()
    grid.header.frame_id = "map"
    grid.header.stamp = rospy.Time.now()

    # Set the grid metadata
    meta = MapMetaData()
    meta.resolution = resolution
    meta.width = width
    meta.height = height
    meta.origin.position.x = -width * resolution / 2  # Center the grid
    meta.origin.position.y = -height * resolution / 2
    meta.origin.position.z = 0.0
    meta.origin.orientation.w = 1.0
    grid.info = meta

    grid.data = flattened_data
    return grid

```

### Step 4/Everything All Together(Added the self.occupancy_pub publisher):

```python
#!/usr/bin/env python

import rospy
import numpy as np
import sensor_msgs.point_cloud2 as pc2
from sensor_msgs.msg import PointCloud2
from nav_msgs.msg import OccupancyGrid, MapMetaData

class PointCloudToOccupancyGrid:
    def __init__(self):
        # Parameters for the occupancy grid
        self.resolution = rospy.get_param("~resolution", 0.05)  # meters per cell
        self.width = rospy.get_param("~width", 100)  # cells
        self.height = rospy.get_param("~height", 100)  # cells
        self.lower_threshold = rospy.get_param("~lower_threshold", 0.1)  # Min Z height
        self.upper_threshold = rospy.get_param("~upper_threshold", 2.0)  # Max Z height

        # Initialize ROS node, subscriber, and publisher
        rospy.init_node('pointcloud_to_occupancygrid', anonymous=True)
        rospy.Subscriber('/lio/map', PointCloud2, self.pointcloud_callback)
        self.occupancy_pub = rospy.Publisher('/map', OccupancyGrid, queue_size=10)

    def pointcloud_callback(self, msg):
        # Convert the PointCloud2 message to a numpy array of points
        points = np.array([p[:3] for p in pc2.read_points(msg, skip_nans=True)])

        # Filter points by height and project to 2D
        points_2d = self.project_to_2d(points)

        # Generate and publish the occupancy grid
        grid = self.generate_occupancy_grid(points_2d)
        self.occupancy_pub.publish(grid)

    def project_to_2d(self, points):
        """Project 3D points to the XY plane, considering a height range."""
        mask = (points[:, 2] >= self.lower_threshold) & (points[:, 2] <= self.upper_threshold)
        filtered_points = points[mask]
        return filtered_points[:, :2]  # Return only X, Y coordinates

    def generate_occupancy_grid(self, points):
        """Generate an OccupancyGrid from a set of 2D points."""
        # Create an empty grid initialized to -1 (unknown)
        grid_data = np.full((self.height, self.width), -1, dtype=np.int8)

        # Map points to grid cells
        for (x, y) in points:
            gx = int((x / self.resolution) + self.width // 2)  # Offset to center of grid
            gy = int((y / self.resolution) + self.height // 2)
            if 0 <= gx < self.width and 0 <= gy < self.height:
                grid_data[gy, gx] = 100  # Mark as occupied

        # Flatten the 2D grid to 1D list
        flattened_data = grid_data.flatten().tolist()

        # Create the OccupancyGrid message
        grid = OccupancyGrid()
        grid.header.frame_id = "map"
        grid.header.stamp = rospy.Time.now()

        # Set metadata
        meta = MapMetaData()
        meta.resolution = self.resolution
        meta.width = self.width
        meta.height = self.height
        meta.origin.position.x = -self.width * self.resolution / 2  # Center the grid
        meta.origin.position.y = -self.height * self.resolution / 2
        meta.origin.position.z = 0.0
        meta.origin.orientation.w = 1.0  # No rotation
        grid.info = meta

        grid.data = flattened_data
        return grid

if __name__ == '__main__':
    try:
        node = PointCloudToOccupancyGrid()
        rospy.spin()
    except rospy.ROSInterruptException:
        pass
```
